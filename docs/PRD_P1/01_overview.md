## 2. 서비스 한 줄 정의

게임 실력 향상을 원하는 학생과 코치를 연결해주는 게임 강의 매칭 웹 플랫폼

## 3. 기획 의도

### 현재 문제점
- 게임 실력을 향상시키고자 하는 사용자들이 체계적인 강의를 찾기 어려움
- 개인 코칭을 받을 수 있는 플랫폼이 제한적
- 코치의 실력·신뢰도를 사전에 확인하기 어려워 강의 선택에 어려움 존재
- 기존 게임 아카데미는 한 달 단위 이상의 수강 방식 → 단발성 코칭 원하는 사용자에게 접근성 낮음

### 해결 방향
- 게임 유저와 코치를 연결하는 강의 매칭 플랫폼 구축
- 코치 정보 및 후기 확인 후 자신에게 맞는 강의 선택 가능
- 단발성 또는 맞춤형 코칭 환경 제공

## 4. P1 범위 (Phase 1)

P1은 **인증 없이 동작하는 핵심 기능 구현** 단계로, 다음을 완성한다.

- 프론트엔드: 전체 UI 구현 (메인, 강의 목록/상세, 마이페이지, 프로필, 로그인/회원가입, 코치 대시보드, 커뮤니티, 강의 콘텐츠)
- 백엔드: REST API 전체 구현 (인증/유저/강의/수강신청/리뷰/커뮤니티/콘텐츠)
- 데이터베이스: MySQL 스키마 설계 및 시드 데이터 삽입
- API 문서: Swagger UI 적용
- Mock 데이터 → 실제 백엔드 API 연동 전환

### P1에서 의도적으로 제외한 항목
- JWT 인증 (P2에서 구현)
- RBAC (역할 기반 접근 제어) — 라우트 보호 없음
- AI 강의 추천
- 학생 성장 분석

## 5. 기술 스택

| 영역 | 기술 |
|------|------|
| 프론트엔드 | React, Zustand, Tailwind CSS, Vite |
| 백엔드 | Node.js, Express |
| 데이터베이스 | MySQL |
| API 문서 | Swagger (swagger-jsdoc, swagger-ui-express) |
| 상태 관리 | Zustand |
| HTTP 클라이언트 | Axios |

## 6. 사용자 역할

| 역할 | 설명 |
|------|------|
| 학생 (student) | 강의를 검색하고 수강 신청하는 사용자 |
| 코치 (coach) | 강의를 개설하고 수강생을 관리하는 사용자 |
