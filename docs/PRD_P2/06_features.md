# GCP P2 — 기능 명세서

P2에서 변경되거나 추가된 기능만 기술한다.
변경되지 않은 기능은 P1 기능 명세서(`PRD_P1/05_features.md`) 참조.

---

## 1. 로그인 (변경)

### P1 → P2 변경사항

| 항목 | P1 | P2 |
|------|-----|-----|
| 서버 응답 | `{ success, data: { ...user } }` | `{ success, data: { token, user } }` |
| 저장 항목 | localStorage user만 | localStorage user + token |
| setUser 호출 | `setUser(res.data)` | `setUser(res.data.user, res.data.token)` |

---

## 2. 라우트 보호 (신규)

### PrivateRoute — 로그인 필요

비로그인 상태로 보호된 경로 접근 시 `/login`으로 리다이렉트.

적용 경로:
- `/mypage`
- `/profile`
- `/community/write`, `/community/edit/:id`
- `/lectures/:lectureId/contents`

### RoleRoute — 코치 전용

비로그인 → `/login` 이동. 로그인 상태이나 role이 coach가 아니면 `/` 이동.

적용 경로:
- `/coach/dashboard`
- `/coach/lecture/new`
- `/coach/lecture/edit/:id`
- `/lectures/:lectureId/manage`

## 3. 수강 신청 (변경)

### P1 → P2 변경사항

| 항목 | P1 | P2 |
|------|-----|-----|
| student_id 전달 | Body에 직접 포함 | JWT에서 서버가 추출 |
| 비로그인 신청 | 클라이언트 단에서만 차단 | 서버에서 401 반환 |
| 역할 검증 | 없음 | 코치가 신청 시 403 |

### 수강 신청 버튼 상태 흐름
```
[비로그인]    → "로그인 후 수강 신청이 가능합니다." 안내 표시
[신청 전]     → "수강 신청" 버튼
[신청 후]     → "검토 중" (비활성)
[승인됨]      → "수강 중" (비활성) + "▶ 강의 수강하기" 버튼 노출
[거절됨]      → "거절됨" (비활성)
```
